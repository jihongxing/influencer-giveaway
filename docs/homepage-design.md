# 主页设计方案 - 极简风格（方案B）

## 设计理念

**极简主义**：大量留白、简洁线条、最小化颜色、清晰层次

## 功能模块设计

### 1. 顶部区域（极简）

**未登录状态：**
```
┌─────────────────────────────────┐
│  [Logo/平台名]          [登录]   │
└─────────────────────────────────┘
```

**已登录状态：**
```
┌─────────────────────────────────┐
│  [头像] 你好，{昵称}    [消息]   │
└─────────────────────────────────┘
```

**设计要点：**
- 单色背景（白色或浅灰）
- 无装饰线条
- 最小化图标
- 清晰的字体层次

### 2. 搜索栏（极简线条）

```
┌─────────────────────────────────┐
│  🔍  搜索活动或主播              │
└─────────────────────────────────┘
```

**设计要点：**
- 细边框（1rpx）
- 圆角（8rpx）
- 浅灰背景（#f8f8f8）
- 无阴影

### 3. 分类导航（极简图标）

```
┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐
│  📦  │  │  👕  │  │  📚  │  │  💄  │
│ 全部 │  │ 服装 │  │ 书籍 │  │ 美妆 │
└──────┘  └──────┘  └──────┘  └──────┘
```

**设计要点：**
- 纯文字或极简图标
- 无背景色（或极浅灰）
- 点击时仅改变文字颜色
- 等间距排列

### 4. 活动列表（极简卡片）

**卡片设计：**
```
┌─────────────────────────────────┐
│  [图片]                          │
│                                  │
│  物品名称                         │
│  运费 ¥XX                        │
│  ────────────────────────────   │
│  [主播头像] 主播昵称              │
└─────────────────────────────────┘
```

**设计要点：**
- 白色背景
- 细边框（1rpx #e0e0e0）
- 圆角（12rpx）
- 无阴影或极淡阴影
- 大量留白（内边距 30rpx）
- 图片占卡片 60% 高度
- 信息区域极简排版

### 5. 快捷操作（浮动按钮 - 仅主播）

```
                    ┌──┐
                    │ +│
                    └──┘
```

**设计要点：**
- 圆形浮动按钮
- 固定右下角
- 仅主播显示（创建活动）
- 极简图标

## 个性化内容策略

### 未登录用户
- **推荐活动**：按热度排序，展示 6-8 个
- **分类导航**：全部、服装、书籍、美妆、其他
- **底部提示**：引导注册/登录

### 已登录主播
- **我的活动**：优先展示自己的活动（前 3 个）
- **推荐活动**：其他热门活动（3-5 个）
- **快捷操作**：浮动"+"按钮（创建活动）
- **顶部提示**：未处理订单数（如有）

### 已登录粉丝
- **推荐活动**：基于浏览历史/兴趣推荐（6-8 个）
- **我的订单**：最近订单快捷入口（顶部）
- **收藏活动**：收藏的活动（如有）

## 视觉设计规范

### 颜色方案（极简）
- **主背景**：#FFFFFF（白色）
- **次背景**：#F8F8F8（极浅灰）
- **文字主色**：#333333（深灰）
- **文字次色**：#999999（中灰）
- **强调色**：#07C160（微信绿，仅用于关键操作）
- **边框色**：#E0E0E0（浅灰）
- **无装饰色**：避免使用过多颜色

### 字体规范
- **标题**：32rpx，粗体，颜色 #333
- **正文**：28rpx，常规，颜色 #333
- **辅助文字**：24rpx，常规，颜色 #999
- **无装饰字体**：系统默认字体

### 间距规范
- **页面边距**：30rpx
- **卡片间距**：20rpx
- **内容内边距**：30rpx
- **留白区域**：40rpx（重要区域之间）

### 圆角规范
- **卡片**：12rpx
- **按钮**：8rpx
- **图片**：8rpx
- **搜索栏**：8rpx

## 交互设计

### 加载状态
- 使用已有的 `loading` 组件
- 极简的加载动画（旋转圆点）

### 空状态
- 大量留白
- 简洁的提示文字
- 无装饰图标

### 错误状态
- 使用已有的 `error-message` 组件
- 极简的错误提示

### 下拉刷新
- 极简的刷新动画
- 无装饰的刷新提示

## 响应式设计

### 不同屏幕适配
- 使用 rpx 单位
- 卡片宽度自适应
- 图片比例固定（16:9）

### 性能优化
- 图片懒加载
- 分页加载（每次 10 个）
- 缓存活动列表

## 技术实现要点

### 1. 用户状态检测
```javascript
// 检查登录状态
const sessionToken = wx.getStorageSync('sessionToken');
if (sessionToken) {
  // 获取用户信息
  const userInfo = await apiService.get('/auth/me');
  // 根据角色显示不同内容
}
```

### 2. 数据获取策略
- **未登录**：获取热门活动（GET /activities/public?sort=popular）
- **主播**：获取自己的活动 + 热门活动
- **粉丝**：获取推荐活动（基于算法或随机）

### 3. 组件复用
- 复用 `loading` 组件
- 复用 `error-message` 组件
- 创建 `activity-card` 组件（可复用）

### 4. 状态管理
- 使用页面 data 管理状态
- 缓存用户信息和活动列表
